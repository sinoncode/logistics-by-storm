# Logistics By Storm - Project Overview

## Project Overview

**Project Name:** Logistics By Storm (Repository: wowdash-react-typescript-shadcn)  
**Type:** Full-stack React dashboard application with TypeScript  
**Purpose:** Multi-feature logistics and business management dashboard with various specialized dashboards and modules  
**Deployment:** Vercel (configured via vercel.json)  
**MVP Status:** This is the Minimum Viable Product (MVP) featuring a comprehensive logistics dashboard with RBAC controls for team member access management.

## Architecture & Design Patterns

### Component Structure
- **Layout Components**: MainLayout, SidebarLayout, Header, Footer, Breadcrumb
- **Page Components**: Organized by feature (dashboards, pages, auth)
- **UI Components**: Reusable shadcn/ui primitives
- **Feature Components**: Dashboard-specific cards and widgets
- **Shared Components**: Logos, theme toggle, navigation

### State Management
- Context API for app-level state (loading, email sidebar, form submission)
- Firebase hooks for auth state
- Local component state for UI interactions
- URL-based routing state (React Router)

### Routing Strategy
- Nested routes with MainLayout wrapper
- Protected routes requiring authentication
- Guest routes redirecting if authenticated
- Error boundaries at route level
- Lazy-loaded page components for code splitting
- 404 fallback route

### Authentication Flow
1. Guest sees login/register/forgot-password pages
2. User registers or logs in via email or OAuth
3. Firebase stores user profile in Firestore
4. Protected routes redirect unauthenticated users to login
5. Session persists via Firebase auth state
6. User can reset password via email

### Styling Architecture
- Tailwind CSS for utility-based styling
- CSS-in-JS via shadcn components
- Dark mode support with next-themes
- Responsive design with Tailwind breakpoints
- Component variants via CVA (class-variance-authority)

### Form Validation
- Zod schemas for type-safe validation
- Custom validation functions for complex rules
- Error state management in context
- React Hook Form for performant form handling
- Email, password, phone, and custom field validators

## Notable Implementation Details

1. **Lazy Loading**: Dashboard pages are lazy-loaded using React.lazy() for better performance
2. **LazyWrapper Component**: Suspense wrapper with fallback loading states
3. **Custom Hooks**: useUsers, useLocalStorage, useIsMobile for common functionality
4. **Theme Management**: next-themes with localStorage persistence
5. **Responsive Design**: Mobile detection at 992px breakpoint
6. **Avatar Generation**: Dynamic avatars using ui-avatars.com API
7. **Form Validation**: Comprehensive Zod schemas with typed errors
8. **Error Boundaries**: Catches React errors at component and route level
9. **Toast Notifications**: Multiple toast library options (react-toastify, sonner)
10. **Firebase Integration**: Full authentication pipeline with email, Google, and GitHub OAuth
